# v2.0-B Tauri 三栏 GUI 实施计划

> **执行说明:** 按 `AGENTS.md` 由单智能体线性执行。T1 是强制 go/no-go 门禁；通过前不进入 T2+。UI 与粘合 code-first 后补特征测试，参数契约、状态栈和安全边界 test-first。不使用子智能体，不自动执行 Git 提交。

**Goal:** 交付可双击运行的 Tauri + React 桌面应用，复用 Python sidecar 完成拖图、调参、diff、收藏与导出闭环。

**Architecture:** Tauri 只管理窗口、sidecar 生命周期和端口注入；React 通过 localhost HTTP 复用现有 `gui/api.py`/`server.py`；Python 引擎仍是参数、渲染、收藏和导出的唯一实现。先完成稳定结构和功能闭环，再在 T9 集中统一布局、间距和视觉。

**Tech Stack:** Python 3.11+、numpy + numba + pyvips、PyInstaller onedir、Tauri、Rust、React + TypeScript + Vite。

## 全局约束

- 先 Windows 验收；T1 失败则停止 Tauri 业务 UI，记录摩擦点并回到作者决策回退方案。
- sidecar 必须是真实引擎：冻结后能 import numba + pyvips、JIT 并渲染真实一帧；不得用 ping stub 代替。
- PyInstaller 优先 onedir；numba cache 必须落在用户可写目录。
- 前端参数范围与默认值只从 `looklift/render/contract.py` 的 API 投影获取，不手抄范围表。
- `analysis` 是面板、渲染、收藏和导出的单一状态对象；`editorStore` 是版本栈唯一 owner。
- 不实现聊天、undo UI、局部调整、批量 UI 或全 RAW 解码。
- 依赖许可必须明确且宽松；不碰 GPL 代码或二进制。
- 自动化测试离线；真机 WebView、Defender、拖拽手感与视觉集中人工验收。
