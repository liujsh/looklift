## 一句话定位(2026-07-17 升级)

**looklift 是开源的一站式 AI 调色应用**——上传照片,在应用内用「大师模板 / AI 推荐」起手,
左侧和 AI 对话调整、右侧像 LR 那样拉滑杆精调、中间实时看效果与原图对比,**不用打开 LR/PS
就能出成片**,也能一键导出 Lightroom 预设 / RAW sidecar 给专业工作流。

核心差异化不变:**白盒 + 教学**——AI 改的永远是可解释的参数(不是黑盒像素),每一步都能看懂、
能微调、能学会。把「调色审美」变成「可复用、可学习、可分享的资产」。

**演进关系(不是推倒重来)**:v0.1-v0.4 已把引擎做到七成(AI 逆向出白盒参数 + 本地渲染 + 风格库
+ XMP 导出),v2 是在这个引擎上加「一站式三栏交互壳 + 成品级渲染 + 应用内编辑」。引擎、供应商层、
CLI、XMP 导出全部 carry forward;只有 pywebview 壳 + vanilla-JS 前端被 Tauri+React 取代。

**对外定位语**(白盒差异化;措辞确保与事实一致——云端模型分析时照片会发给该服务商):
> 别的 AI 修图给你黑盒结果,looklift 给你白盒过程——参数可改、原理可学、模板可攒。
> 调色、渲染、风格库全在你电脑上;AI 只把照片发给你自己选的模型,接本地模型(Ollama)可彻底离线。

市场洞察:参考图取色流派(Color.io)只出黑盒 LUT;风格学习流派(Imagen/Aftershoot)只卖"省时间";
一站式黑盒生图(扩散类)改的是像素不可控;LR 伴侣类(JarvisArt)要求你有 LR。
**"AI 白盒调参 + 应用内成品 + 教学讲解"三合一,且 Python 里逆向 LR 滑杆这个位置无人占据——是我们独占的。**

## 白盒双向:一套参数,两条出口(核心价值主张,勿在迭代中丢失)

同一套可解释白盒参数(ANALYSIS_SCHEMA),用户在 looklift 里(拖滑杆 / 跟 AI 聊天 / 选模板)调好后,
**两条出口自由选,不二选一、不锁定**:

| 出口 | 面向 | 交付 |
|---|---|---|
| **应用内出片** | 想轻量方便、不开 LR 的爱好者 | 三栏应用内调色 + 成品级渲染 → 直接导出成片(嵌 ICC) |
| **导出给专业工具** | 追求极致、要在 LR/PS/达芬奇精修的用户 | 导出 LR 预设 / RAW sidecar / .cube LUT → 进专业管线用其引擎出片 |

这是相对所有竞品的独占优势:JarvisArt 必须有 LR;扩散生图是黑盒不可导不可改;纯 LUT 工具黑盒不可改。
**looklift 的参数全程白盒、双向可用**——这条主张必须贯穿 v2 所有迭代,渲染和导出永远读同一份参数。

## 目标用户(按优先级)

| 画像 | 描述 | 需要什么 |
|---|---|---|
| **P1 入门爱好者**(最大人群,作者本人) | 想要好看的调色但不懂参数,不想学 LR | 应用内三栏、拖拽、AI 对话、模板起手、中文讲解——**app 内轻量出片** |
| **P2 进阶玩家** | 有明确风格追求,在乎还原度和可控 | 逐参数面板精调、批量、refine 闭环、风格库、**导出给 LR 精修** |
| **P3 技术型用户** | 会命令行,要自动化 | CLI(已有)、可脚本化、CI 可跑 |

## 核心价值线(v2 升级后)

1. **像(还原度)** ✅ 引擎已有——本地渲染预览、refine 自动闭环、还原度评分。v2 升级为**成品级渲染**
   (线性光 + 色彩管理 + numba 实时),从"方向近似"到"应用内直接出片"。
2. **库(风格资产)**——风格收藏、检索、批量分析、**内置大师模板库**(青橙/胶片/日系)、分享。
3. **懂(调色教学)**——AI 改的是可解释参数 + 报告即调色课 + 每个模板附拆解。**白盒是"懂"的地基。**
4. **调(应用内编辑)** 🆕——三栏交互:左 AI 对话调参、中画布看效果/diff、右 LR 风格滑杆面板精调。
   这是 v2 新增的第四条线,把前三条从"给 LR 用"升级成"应用内闭环"。

## 最终形态(v2 一站式)

**独立的一站式 AI 调色应用(Open Design 三栏形态)**:
- **左**:AI 聊天区——自然语言调色("天空再蓝一点"),AI 输出参数 delta、渲染、看 diff、多轮自修正
- **中**:画布——实时预览 + 与原图 before/after 对比;上传照片、选模板起手
- **右**:LR 风格调色面板——逐参数滑杆(基础/HSL/曲线/颜色分级/效果),手动精调,与 AI/模板同步
- **图库**:用户自建 + 内置大师模板库;手动传图(不做抓取)
- **输出**:应用内导出成片(嵌 ICC),或导出 LR 预设 / RAW sidecar 给专业工作流(白盒双向)
- **技术形态**(2026-07-17 锁定):**Tauri 壳 + React 前端 + Python 引擎 sidecar**;打包成桌面安装包
- 开源(MIT,全程许可干净),面向摄影/修图社区发布

> 旧「最终形态 v1.0」(pywebview 单窗口 LR 伴侣)已被 v2 一站式取代;v0.1-v0.4 作为达成 v2 的引擎地基保留。

## 功能模块总览(v1.0 全景)

三层九模块;核心原则:引擎层是唯一的功能实现,CLI 与 GUI 都只是入口。

**核心引擎层**
| # | 模块 | 功能 | 落点 |
|---|---|---|---|
| 1 | AI 分析引擎 | 单图/原片对照/多图归纳 → 参数+中文讲解;多供应商(CC/API 已有,中转/Ollama v0.5) | analyzer, providers |
| 2 | 参数读写 | 读 JPEG 内嵌 LR 元数据(100% 精确);写预设 .xmp / RAW sidecar | xmp_reader/writer ✅ |
| 3 | 渲染与评分 | preview 本地近似渲染;还原度 0-100;.cube LUT 导出 | render, v0.3 |
| 4 | 精修闭环 | refine 手动(✅)+ auto 自动迭代到收敛(v0.3) | refine |
| 5 | 风格库 | 收藏/套用/批量(✅);强度滑杆(v0.4);预置 10 经典 look(v0.6);目录聚类(v0.5+) | looks/ |
| 6 | 报告与教学 | HTML 报告(✅);"调色课"式讲解+look 拆解课(v0.6)——「懂」独占赛道 | report |

**界面层**:7. CLI(✅,全功能可脚本化)| 8. GUI(v0.4,pywebview,Claude 风 tokens,
拖拽→分析→报告→导出、库面板、对比条、首次配置向导)

**分发层**:9. pip(✅)+ Windows 单 exe(v0.7);Mac 走 pip

主流程:照片 → AI 逆向白盒参数 → 本地渲染验证 → 自动精修 → 入风格库 →
出预设/LUT/报告课。价值线归属:「像」=3+4,「库」=5,「懂」=6。

## 非目标

- 不做云服务/账号体系(保持本地工具,零运营成本)
- **不做黑盒生图/扩散式像素改写**——AI 只改可解释参数,白盒是产品红线(这是与一站式黑盒工具的核心区别)
- **不驱动真 Adobe LR**(JarvisArt 那套)——要求用户装 LR、平台锁死、打不了独立包,与"一站式独立应用"矛盾
- v2 一站式起**先只做全局参数**;**局部调整(蒙版/渐变)列 v2.x 远期**(需分割模型),第一版一站式不做
- **RAW 支持策略(2026-07-17 调研定案,U28)**:走**轻量方案 = 抠 RAW 内嵌的相机 JPEG 预览**
  (rawpy `extract_thumb`,一个库 LibRaw 统一上千相机型号,我们不写每相机代码;预览是相机自身渲染的
  sRGB 图,直接进现有管线,~1 天工作量,守 sRGB 边界)。**不做全 RAW 解码 + 每相机色彩 profile 匹配**
  (那是 RAW 冲洗器领域,无底洞;我们是调色工具)。可早至 v2.0-B 图库上传(拖 RAW = 拖张图);
  「全解码 + 14bit 高光宽容度」作为 v2.x **可选高画质开关**再议。许可:LibRaw 取 CDDL(宽松不传染)+ 署名。
  **画质取舍(诚实记录)**:内嵌预览是相机全分辨率自渲染 JPEG(8bit)——对典型调色画质足够、基本无感;
  真实牺牲只在**极端曝光抢救**(8bit vs 14bit 断层、死白高光救不回),那类需求走 v2.x 的全解码开关。
  且预览**更契合我们的参数模型**:LR 风格参数假设起点是已渲染图,预览正是(和现有 JPEG 输入同类),
  而全解码给的是平的线性 RAW + LibRaw 默认色(不像 LR),套我们参数结果反而偏。取预览时优先取最大那张;
  个别老/入门机身预览偏小,文档注明。
- 打包:v2 桌面安装包优先 Windows;**macOS 保持核心兼容**(Python 引擎 + Tauri 天然跨平台,CI 含 macOS);
  苹果签名($99/年)视社区需求再议
- v2.0 前不做 Capture One / DxO 等其他软件格式
- v2.0 前不做多语言/i18n:界面/讲解/文档全中文(目标用户与发布渠道都在中文圈),README 附英文简介段
- **技术栈不走回头路**(2026-07-17 锁定):引擎 Python(numpy+numba+pyvips,不换 Rust/C);壳 Tauri+React;
  全程许可干净不碰 GPL 代码(只学架构不抄码)。选型依据见开发计划的调研 + spike 实证。
