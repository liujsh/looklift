# v2.2 需求：平台外壳

> 状态：已确认（2026-07-20，作者逐段确认范围、状态边界与验收方式）。
> 上游：[产品需求](../../product/requirements.md) · [平台化 UI 设计](../../product/platform-ui-design.md)。
> 同迭代：[设计](./design.md) · [任务](./tasks.md)。
> 前置：v2.1 AI Studio 已完成人工验收并合入 `main`。

## 一句话目标

把现有单页 AI Studio 迁入可持续扩展的平台外壳：用户从首页快速打开照片或恢复最近正式会话，
在多个互相隔离的 Studio 标签间切换，并始终清楚哪些入口属于后续版本。

## 范围

| 本迭代做 | 本迭代不做 |
|---|---|
| 常驻可折叠全局导航、固定首页标签和多个 Studio 标签 | 文件夹索引、缩略图数据库、标签和完整图库（v2.3-A） |
| 首页“快速修图”和“继续修图”真实可用 | 相机、存储卡或手机发现与复制导入（v2.3-B） |
| “添加文件夹”“从设备导入”可见并明确标注后续版本 | 用假数据伪造最近图库、设备或自动化结果 |
| 每个 Studio 独立保存编辑、聊天、请求和临时预览内存状态 | 标签拖拽排序、固定、跨窗口或复杂工作区恢复 |
| 最近正式会话列表、源文件可用性和主动恢复 | 持久化未确认预览，或启动时自动进入最后会话 |
| Tauri 原生文件选择；浏览器开发模式保留上传回退 | 扩展完整照片资产、内容指纹或文件重新定位模型 |
| 现有 AI Studio 原样迁入平台，补齐标签关闭门禁 | 重写 AI、画布、直方图、参数面板或渲染业务 |
| 图库、模板、自动化、插件和设置的统一版本说明页 | 模板教学、自动化执行、插件目录和任意第三方代码 |

## 用户场景

- 我启动 LookLift 后先看到明确的开始入口，而不是被直接送进上一次工作台。
- 我可以从本机选择一张支持的照片快速修图，原文件仍由我管理。
- 我可以从首页继续最近一次已经确认的正式调色状态。
- 我可以同时打开多个 Studio 标签，切换后各自的照片、参数、聊天和 AI 状态不串。
- 我离开或关闭带未确认 AI 预览的标签时，系统不会静默保存或丢弃候选。
- 我能看到未来的图库和设备入口，但不会误以为 v2.2 已经实现这些能力。

## 验收标准

### 平台外壳与导航

- [ ] 应用连接引擎后进入首页；首页标签固定存在且不可关闭。
- [ ] 全局导航包含首页、我的图库、大师模板、自动化技能、插件、设置与帮助。
- [ ] 普通平台页面沿用用户上次导航折叠状态；Studio 首次进入默认折叠为图标轨。
- [ ] 未实施页面只展示能力说明和目标版本，不调用不存在的业务接口或制造假数据。
- [ ] 对外品牌统一写作 **LookLift**，包、仓库和配置键继续使用 `looklift`。

### 首页与快速修图

- [ ] 首页主入口保持“添加文件夹 / 从设备导入 / 快速修图”三种心智。
- [ ] 前两个入口明确标注 v2.3；点击只解释版本边界，不启动扫描或设备访问。
- [ ] “快速修图”在 Tauri 中使用系统文件选择器取得真实路径，并打开或聚焦对应 Studio。
- [ ] 浏览器开发模式继续支持现有上传回退；临时文件丢失时诚实标记不可恢复。
- [ ] 用户取消选择或打开失败时留在首页，显示中文可恢复提示，不创建空白标签。

### 标签与 Studio 隔离

- [ ] 可同时打开多个 Studio 标签；同一个正式 session 不重复打开。
- [ ] 每个标签的 analysis、强度、聊天、AI 请求、临时预览、直方图和面板状态互相隔离。
- [ ] 切换标签不提交或丢弃临时预览；后台 AI 结果只能回到发起请求的标签。
- [ ] 非活动 Studio 不响应窗口级拖放等全局输入。
- [ ] 关闭普通标签会释放请求、监听器和预览资源，不影响其他标签。

### 最近会话与恢复

- [ ] 首页按最后活动时间展示有限数量的最近正式会话。
- [ ] 会话卡片只使用已持久化的正式状态，不暗示临时预览能够跨重启恢复。
- [ ] 点击可用会话后载入照片、当前正式 analysis 和正式聊天记录，再生成效果预览。
- [ ] 源文件缺失时保留会话记录并标记“源文件不可用”，不自动搜索、复制或替换文件。
- [ ] 最近会话查询复用 v2.1 SQLite 表，不提前建立 v2.3 图库实体或迁移。

### 关闭门禁与异常

- [ ] 关闭含临时预览的标签时提供“保留并关闭 / 放弃并关闭 / 取消”。
- [ ] 关闭正在运行 AI 请求的标签前要求停止；晚到响应不能更新已销毁状态。
- [ ] 会话读取、源文件检查或预览失败不生成半恢复状态，并提供重试或返回首页出口。
- [ ] 应用退出后只恢复最后正式版本，未确认候选永不偷偷入库。
- [ ] v2.1 的 AI 编辑锁、异步竞态保护、当前效果代理图和直方图行为不回归。

### 桌面布局

- [ ] 约 1440px 宽度可同时展示导航轨、聊天、画布和参数面板。
- [ ] 约 1100px 宽度保持画布可用并正确应用折叠状态。
- [ ] 900px 最小窗口不继续压缩为手机布局；过窄时给出明确提示。
- [ ] 纯视觉正确性以真实 Tauri/浏览器渲染检查为准，不为像素值编写脆弱测试。

## 非目标

- 不做云端账号、照片托管、跨设备同步或多人协作。
- 不改变白盒参数模型，不引入黑盒生图、局部蒙版或 RGB 分通道曲线。
- 不在 v2.2 实现本地图库、设备复制导入、RAW、模板教学、自动化或插件市场。
- 不做源文件移动、重命名、删除、重新定位或内容指纹恢复。
- 不持久化标签布局、面板内存状态或未确认预览；正式会话仍是唯一跨重启恢复边界。

## 完成条件

上述契约无矛盾，自动化测试和真实 Windows/Tauri 人工验收通过，现有 Studio 行为不回归，且未把
v2.3 及后续能力提前塞入平台空壳。
