# v2.2 设计：平台外壳

> 状态：已确认（2026-07-20，作者逐段确认）。
> 同迭代：[需求](./requirements.md) · [任务](./tasks.md)。
> 视觉基线：`.superpowers/brainstorm/397-1784352572/content/` 中的平台外壳、首页和 Studio 原型；
> 原型只作视觉与信息架构参考，版本边界以本文为准。

## 1. 核心决策

### D1：平台层负责“去哪里”，Studio 负责“怎么修”

平台层只拥有全局导航、标签和首页。现有 Studio 继续拥有 AI 对话、画布、白盒参数与底部模板带；
调色数学、参数契约和持久化业务不复制到平台组件。

### D2：首页固定，多 Studio 标签使用独立运行时

首页标签固定存在。每个 Studio 标签关联一套独立编辑 Store、会话协调器、聊天工作流和界面状态。
标签切换只改变可见性，不把多个照片塞进同一个全局 Store，也不通过切图重置来假装多标签。

### D3：非活动 Studio 保持内存状态，但不能接收全局输入

为了保留聊天消息、折叠状态、直方图、候选预览和进行中的 AI 请求，未关闭的 Studio 在标签切换后保留
运行时。只有活动 Studio 可以注册窗口级拖放等全局事件。关闭标签时显式释放请求、监听器、对象 URL 和
其他派生资源，避免后台状态泄漏。

### D4：启动回到首页，恢复由用户主动触发

应用连接引擎后总是显示首页。首页列出最近正式会话，用户点击后才恢复。该选择保留平台入口心智，避免
启动时自动进入已经移动、删除或不再想继续的照片。临时预览仍只存在内存，重启后不存在。

### D5：正式桌面选择真实路径，浏览器上传只是开发回退

Tauri 使用官方文件对话框取得原文件路径，保证正式会话能在重启后引用用户管理的文件。普通浏览器拿不到
稳定绝对路径，继续复用现有上传临时文件；恢复前检查文件是否存在，丢失时诚实降级，不把临时上传升级成
图库资产。

### D6：最近会话是 v2.1 表上的只读投影

最近会话查询只从现有会话、当前版本指针和版本表投影摘要，不增加照片资产、索引根、标签或内容指纹表，
因此不需要数据库迁移。完整恢复仍使用现有单会话读取契约。

### D7：未来入口可发现，但不得伪装完成

首页保留三种开始方式，全局导航保留最终信息架构。未进入本期的入口统一显示目标版本与能力边界，不展示
假的最近图库、设备照片、执行记录或插件目录，也不触发任何隐藏扫描。

### D8：关闭是显式状态机

普通正式标签可直接关闭；存在临时预览时必须保留、放弃或取消；AI 请求运行中必须先停止。关闭动作只有在
对应异步操作完成后才能销毁运行时，晚到响应继续通过活动请求身份和运行时存活状态双重拒绝。

## 2. 状态所有权

```text
PlatformShell
├─ navigationState：当前顶层页面、导航折叠偏好
├─ tabRegistry：标签顺序、活动标签、关闭状态
└─ StudioRuntime × N
   ├─ editorStore：analysis、强度、版本、pendingPreview、AI 锁
   ├─ sessionCoordinator：session ID 与正式写入队列
   ├─ chatWorkflow：消息、请求序号、取消控制
   └─ viewState：面板折叠、直方图和其他仅内存派生状态
```

平台 Store 不读取或改写 analysis。Studio 运行时不决定全局路由。两者只通过“打开、聚焦、请求关闭、
销毁”这些有限动作协作。

## 3. 标签模型与生命周期

标签分为：

- `home`：固定、唯一、不可关闭。
- `platform`：未来页面的说明标签；同一页面只保留一个。
- `studio`：绑定一个正式 session 和独立运行时。

新建或恢复 session 后，平台先检查是否已有相同 session 的标签；有则聚焦，没有才创建。非活动 Studio
保持挂载但对用户不可见，并暂停窗口级输入。标签关闭后不能再接收 provider、预览或持久化回调。

标签顺序和活动标签不跨重启持久化。跨重启的唯一真相仍是 SQLite 中的正式 session；这样不会把窗口工作区
偏好误当成业务恢复数据。

## 4. 首页

首页沿用已确认原型的“今天想修哪组照片？”层级，但 v2.2 只展示真实内容：

1. **添加文件夹**：标注 v2.3-A，点击说明“只索引、不复制”的未来语义。
2. **从设备导入**：标注 v2.3-B，点击说明设备复制与 RAW 门边界。
3. **快速修图**：打开文件选择器，成功后进入 Studio，不加入图库。
4. **继续修图**：展示最近正式会话；不展示“临时预览已恢复”等错误文案。

“最近图库”“推荐模板”和“快捷自动化”只有在对应版本存在真实数据和动作后再加入，本期不使用示例卡片占位。

## 5. 最近会话契约

新增最近会话只读端点，接收有上限的 `limit`，按最后活动时间降序返回摘要。摘要包含恢复所需的 session
身份、供首页显示的文件名与正式版本摘要、更新时间和源文件是否可用；不返回完整 analysis 或临时状态。

恢复流程：

```text
首页请求最近会话
→ 用户点击可用卡片
→ 若标签已打开则聚焦
→ 否则读取完整 session snapshot
→ 创建独立 StudioRuntime
→ 恢复正式 analysis 与聊天
→ 生成当前效果预览
```

源文件缺失时卡片仍保留，但恢复按钮禁用。v2.2 不猜测新路径；v2.3 再通过文件身份和内容指纹提供恢复能力。

## 6. 快速修图数据流

```text
点击快速修图
→ Tauri 原生文件对话框（浏览器开发模式走上传回退）
→ 用户取消：无状态变化
→ 取得路径并创建或恢复现有 session
→ 去重并打开 Studio 标签
```

支持格式继续沿用现有 JPEG、PNG、WebP 和 TIFF 边界。平台层不重新解析图片或建立另一套格式白名单；
最终校验仍由既有本地服务负责。

## 7. Studio 迁入

现有 `EditorShell` 从直接引用模块级 Store 改为消费所属运行时。迁移只改变依赖注入和外层布局：

- AI、画布、面板、直方图和模板继续通过同一 analysis 工作。
- 每个 Canvas 仅在所属标签活动时处理原生拖放。
- 隐藏标签中的 AI 请求可以完成，但结果只能落到其运行时。
- 原有编辑锁、预览签名和请求 ID 校验全部保留。

不借 v2.2 重构渲染管线、参数面板或聊天契约。

## 8. 关闭状态机

```text
请求关闭 Studio
├─ AI 请求运行中 → 提示停止 / 取消关闭
├─ pendingPreview 存在 → 保留并关闭 / 放弃并关闭 / 取消
└─ 仅正式状态 → 直接关闭
```

“保留并关闭”必须等待正式事务成功；失败时标签继续存在并显示错误。“放弃并关闭”只丢弃内存候选，不回退
或删除之前的正式版本。运行时销毁前统一取消仍在途的预览与 provider 请求。

## 9. 异常处理

| 场景 | 行为 |
|---|---|
| 文件选择取消 | 留在首页，不显示错误、不创建标签 |
| 文件类型或读取失败 | 首页显示中文提示，可重新选择 |
| 最近会话查询失败 | 首页其他入口仍可用，最近会话区提供重试 |
| 源文件已移动或删除 | 保留摘要并标记不可用，不自动搜索 |
| 完整 session 恢复失败 | 不创建半初始化标签，留在首页重试 |
| 恢复后首次预览失败 | 保留正式状态，Studio 内允许重试或关闭 |
| 切换标签时旧预览晚返回 | 只按所属运行时和签名更新，不影响活动标签 |
| 关闭时正式提交失败 | 不销毁标签，保留候选并显示保存失败 |
| 已关闭标签收到异步结果 | 运行时存活检查拒绝结果，不重建标签 |

## 10. 响应式与视觉

复用原型的暖色纸张平台层和深色照片舞台，不改变现有 LookLift 调色工作台的视觉语言。Studio 的全局导航
默认折叠；普通页面可展开文字。约 1440px、1100px 和最低 900px 是人工验收断点，具体像素以真实渲染
调试为准。低于最小宽度时提示增大窗口，不继续压缩成手机布局。

纯间距、颜色、圆角、阴影和字号不写断言；标签状态、折叠逻辑、条件渲染和事件归属使用行为测试。

## 11. 测试策略

### Python

覆盖最近会话的排序、数量上限、正式版本摘要、文件存在状态和 HTTP 参数错误。所有测试继续使用隔离 home、
临时数据库和临时照片，不触网、不读取真实用户配置。

### React

覆盖首页固定标签、多运行时隔离、重复 session 聚焦、标签切换保留、非活动标签拒绝全局拖放、最近会话恢复、
快速修图取消/失败和关闭状态机。现有 Studio 测试继续验证 AI、画布、直方图与参数行为。

### 人工

真实 Windows/Tauri 下用两张照片验证文件选择、多标签切换、后台 AI、临时预览关闭门禁、重启恢复和三个桌面
宽度。纯视觉调整使用 `pnpm dev` 检查，不反复运行全量测试。

## 12. 风险控制

- **全局单例串状态**：Store、协调器和工作流全部改为按 Studio 创建，平台层不直接持有 analysis。
- **隐藏组件重复监听**：窗口级监听显式受活动标签控制，切换和销毁都有生命周期测试。
- **临时上传不可恢复**：正式 Tauri 使用真实路径；浏览器回退恢复前检查存在性。
- **范围滑向图库**：最近会话只投影已有表，不增加照片资产、标签、指纹或扫描任务。
- **视觉挤压画布**：Studio 默认窄轨，并在三个目标宽度人工验收。

## 13. 完成条件

需求、状态所有权、恢复边界、关闭门禁和后续版本范围互相一致即停止设计扩写；代码级细节由同版本实施计划
和测试确定。
