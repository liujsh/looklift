# v2.1 任务：AI Studio

> 状态：已确认（2026-07-18，作者完成书面 review）。
> 同迭代：[需求](./requirements.md) · [设计](./design.md)。
> 仅在作者确认三文档并由 v2.0 完成 M1-M8 收口后生成实施计划。

## T1 参数变更契约

- [x] 从统一参数契约生成 AI 可写白名单，支持标量 delta/赋值。
- [x] 新增主明度曲线整组控制点操作与原子校验；明确拒绝 RGB 分通道和局部参数。
- [x] 覆盖未知路径、越界值、非法曲线、空变更与其余字段不变测试。
- 验收：所有合法变更确定性落到 analysis，非法输入不会形成部分修改。

## T2 AI 对话核心

- [ ] 实现纯 `chat_step` 流程，复用现有 provider 抽象，不在 provider 内保存会话状态。
- [ ] 定义结构化响应：参数操作、中文理由、能力限制和手动操作路径。
- [ ] 使用 mock provider 覆盖单轮、失败、取消、非法响应和无有效变更。
- 验收：相同输入与 mock 响应得到相同规范化结果，全程不触网。

## T3 安全代理图与元数据

- [x] 统一生成最长边 2048px、重新编码且不携带 EXIF 的 AI 代理图。
- [x] 实现默认安全元数据白名单和用户关闭开关；黑名单字段永不进入 provider 请求。
- [ ] 在请求界面显示供应商、代理图数量和元数据发送状态。
- 验收：测试检查输出图无元数据，provider 捕获参数中只出现允许字段。

## T4 临时预览状态

- [ ] 在编辑 Store 增加内存 `pendingPreview`，与正式版本明确分离。
- [ ] 接通保留、撤销、继续手调、切换照片和关闭会话的状态转移。
- [ ] 预览渲染支持取消和请求序号，拒绝过期响应。
- 验收：AI 预览不会提前写版本；所有出口回到明确的正式状态。

## T5 左侧对话界面

- [ ] 完成消息流、输入框、附件入口、参数变化卡片和错误/进度状态。
- [ ] 支持面板常驻可折叠，并与现有画布、右侧参数和底部照片带协调。
- [ ] 暴露“保留此版本”“撤销”和“AI 精修”明确动作。
- 验收：用户无需理解 provider 内部即可完成一次建议—预览—决策流程。

## T6 按需 AI 精修

- [ ] 普通消息固定单次调用；“AI 精修”最多再执行两轮。
- [ ] 支持达成、无变更、取消、超限停止和逐轮进度。
- [ ] 每轮以最新临时预览为输入，但不自动写正式版本。
- 验收：mock 下轮数和停止原因可确定断言，不存在后台无限调用。

## T7 最小 SQLite 持久化

- [ ] 建立 schema version、会话、消息、版本和当前正式版本指针的最小表。
- [ ] 正式消息与版本使用事务写入，临时预览不入库。
- [ ] 增加迁移前最近三份轮换备份和只读恢复入口。
- [ ] 覆盖重启恢复、事务回滚、迁移和临时预览不恢复测试。
- 验收：崩溃或重启后只恢复最后确认状态，不出现半版本。

## T8 能力边界与中文错误

- [ ] 对局部调整、RGB 曲线等不支持请求返回限制、近似方案和右面板操作说明。
- [ ] 统一鉴权、超时、服务未启动、取消、渲染和保存失败提示。
- 验收：失败不改正式版本，用户始终有重试、撤销或手调出口。

## T9 集成与文档回填

- [ ] 完成消息→参数→临时预览→保留/撤销→重启恢复的离线集成测试。
- [ ] 回归现有手调、模板、渲染、导出和 provider 测试。
- [ ] 实现后回填 `docs/design.md` 与 `docs/dev-log.md`，版本号对齐 2.1.0。
- 验收：自动化测试全绿，架构实况与实现一致。
