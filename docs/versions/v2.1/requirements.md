# v2.1 需求：AI Studio

> 状态：已确认（2026-07-18，作者完成书面 review；补充当前效果 AI 输入与直方图）。
> 上游：[产品需求](../../product/requirements.md) · [平台化 UI 设计](../../product/platform-ui-design.md)。
> 同迭代：[设计](./design.md) · [任务](./tasks.md)。
> 前置：v2.0-A 与 v2.0-B 完成 M1-M8 人工验收并共同收口为 2.0.0。

## 一句话目标

把现有三栏编辑器升级为可恢复的 AI Studio：用户发一句自然语言请求，AI 返回可解释参数建议并先形成
临时预览；用户可以保留、撤销或继续手调，AI、模板和手动调整始终共享同一参数真相源和版本历史。

## 范围

| 本迭代做 | 本迭代不做 |
|---|---|
| 左侧 AI 对话、参数 delta、临时预览、保留/撤销 | 平台首页、全局导航和标签（v2.2） |
| 标量参数与主明度曲线整组控制点 | RGB 分通道曲线、局部蒙版、生成式像素修改 |
| 普通消息一次建议；用户主动“AI 精修”最多追加两轮 | 默认自动连续三轮或无限自循环 |
| 正式对话、版本和当前照片状态的最小 SQLite 持久化 | 完整本地图库、标签、设备导入（v2.3） |
| 2048px 代理图与安全元数据白名单 | 上传原始 RAW、原尺寸照片或完整 EXIF |
| 当前效果 RGB 直方图、拍摄信息与编辑基线分层 | 导入时隐式执行自动修图或伪造非零初始调整量 |
| AI 请求期间锁定编辑并允许停止、查看和原图对比 | 首版并发合并用户手调与 AI 返回参数 |
| 能力不支持时给出右侧面板的手动操作路径 | 假装已完成当前引擎不支持的操作 |

## 用户故事

- **U24**：我用大白话描述目标，AI 调整可解释参数，我能看到哪些滑杆或曲线发生了变化。
- **U25**：普通建议不满意时，我可以主动让 AI 查看当前效果并再精修，且轮数可控。
- **U29**：AI 结果先预览，不立即污染正式版本；我可以保留、撤销或接着手调。
- **U30**：重启应用后，正式会话、对话和版本仍能恢复；未确认的临时预览不恢复。

## 验收标准

### 白盒参数与确定性

- [ ] `chat_step` 只返回结构化参数变更和中文解释，不返回黑盒成片。
- [ ] 标量 delta 基于当前值应用，未知路径丢弃、越界值按参数契约处理，其他字段保持不变。
- [ ] 主明度曲线作为整组有序控制点替换并验证端点、顺序、范围和最少点数；非法曲线不落地。
- [ ] AI 修改后，右面板与画布读取同一份当前 analysis；手动修改后再对话也以手改结果为基线。
- [ ] 曝光、高光等编辑参数初始为中性调整量；EXIF、色彩空间、配置文件和拍摄白平衡作为独立基线信息展示，不混入 analysis。
- [ ] 所有自动化测试使用 mock provider，不触网、不调用真实模型。

### 临时预览与版本

- [ ] AI 建议渲染后进入临时预览，不立即写正式版本或数据库。
- [ ] “保留此版本”把临时状态原子写入版本历史；“撤销”恢复上一个正式版本。
- [ ] 用户在临时预览上继续手调，视为接受该预览并从它继续形成正式编辑历史。
- [ ] 切换照片、关闭会话或退出应用时，对未确认预览明确提示保留或放弃。
- [ ] AI、模板和手调共用 undo/redo 与版本栈，不建立互相隔离的历史。

### 对话与按需精修

- [ ] 普通消息默认只调用一次 AI；不会在用户无感知时自动消耗多轮额度。
- [ ] 用户点击“AI 精修”后，AI 可基于当前代理预览追加最多两轮修正，并随时可取消。
- [ ] 每轮 AI 看到的代理图由同一轮 `current_analysis` 和强度渲染得到，不得继续发送与当前参数不一致的原始画面。
- [ ] AI 请求期间锁定右侧参数、模板、强度、重置和历史操作；用户仍可查看画布、拉动原图对比并停止请求。
- [ ] 请求停止或照片切换后，晚到响应按活动请求 ID 与照片身份丢弃，不产生候选或覆盖当前状态。
- [ ] 达成目标、无有效变更、用户取消或达到上限时停止，始终保留最后一版可用临时预览。
- [ ] 对话流逐步展示参数从旧值到新值的变化、简短理由、供应商和本次发送的代理图数量。

### 持久化与隐私

- [ ] SQLite 保存正式会话、消息、版本关系和当前版本；事务失败不得留下半版本。
- [ ] 重启后可恢复正式状态；临时预览只存在内存，不在崩溃恢复后冒充已确认版本。
- [ ] 数据库带 `schema_version`，迁移前创建轮换备份；本迭代只覆盖会话所需最小表。
- [ ] 外部 AI 只接收最长边不超过 2048px 的重新编码代理图，图片本身不携带 EXIF。
- [ ] AI 代理图是按当前参数渲染的临时文件，仅供本轮模型分析；不进图库、不替换原片、不作为导出文件，请求结束后清理。
- [ ] 默认仅以结构化文本发送 ISO、快门、光圈、焦距、曝光补偿、白平衡和色彩空间；用户可关闭。
- [ ] GPS、拍摄时间、设备序列号、作者、版权、完整路径、文件名、内部 ID 和 MakerNotes 永不发送。

### 能力边界与错误

- [ ] 对局部调整、RGB 分通道曲线等暂不支持请求，AI 明确说明限制并指向可操作的右侧面板步骤。
- [ ] 鉴权、超时、模型不可用、取消和非法响应均不改变正式版本，并提供中文可恢复提示。
- [ ] v2.0 的手动编辑、渲染、导出和现有 provider 行为不回归。

### 当前效果直方图

- [ ] 右侧面板显示当前成功预览的 256 档 RGB 直方图，并给出阴影与高光裁切提示。
- [ ] 直方图从当前效果预览计算，而不是从 analysis 推算；调参期间保留上一结果并标记更新中。
- [ ] 只接受与当前预览签名匹配的计算结果，快速连续调参时旧结果不得覆盖新结果。
- [ ] 直方图计算失败不阻断修图，界面给出可恢复占位状态。

## 非目标

- 不在 v2.1 建立文件夹索引、照片标签、设备发现或复制导入。
- 不上传 RAW 原文件，不承诺 AI 直接分析传感器数据。
- 不实现社区模板、自动化技能或第三方插件市场。
- 不实现局部画笔、渐变、选区、修复或生成式编辑。

## 完成条件

上述契约无矛盾，自动化测试通过，并由作者完成真实照片下的临时预览、保留/撤销、重启恢复与隐私提示人工验收。
