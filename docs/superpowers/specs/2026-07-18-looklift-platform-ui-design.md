# LookLift 平台化 UI 与本地图库设计

> 状态：已确认（2026-07-18，作者逐段确认）。  
> 性质：跨迭代产品设计提案，不直接充当某一版本的实施 spec。  
> 交互参照：[Open Design](https://github.com/nexu-io/open-design) 的平台外壳与工作区心智，
> 以及 Lightroom 的导入、图库和修改照片工作流。只借鉴信息架构与交互模式；视觉资产与实现代码
> 不直接复制，后续仍须按 LookLift 自身品牌和许可要求落地。  
> 实施约束：真正开发前，必须先把本提案拆入 `docs/specs/<版本>/` 的
> `requirements.md`、`design.md`、`tasks.md`，并同步产品路线图。

## 1. 结论

LookLift 的最终产品形态采用 **Open Design 式平台外壳 + Lightroom 式修图 Studio**：

- 平台层提供首页、我的图库、大师模板、自动化技能、插件和设置。
- 修图 Studio 提供左 AI 对话、中照片画布、右白盒参数和底部本次照片带。
- 图库不是托管资产库，而是增强型本地文件夹浏览器；原文件始终由用户管理。
- 用户可以完全手调、询问 AI、直接套用模板，或运行自动化技能；所有路径最终只修改同一份
  可解释白盒参数，并进入同一个版本历史。
- AI 建议先成为可撤销的临时预览；用户保留或继续手调后才形成正式版本。
- 模板、自动化和插件有明确边界，任何扩展都不得绕过核心参数模型做黑盒像素改写。

## 2. 目标与非目标

### 2.1 目标

1. 让首次用户从首页即可完成添加文件夹、设备导入或临时快速修图。
2. 让本地文件夹用户获得更好的照片预览、标签搜索、调色版本与导出记录，同时不改变其文件整理习惯。
3. 让专业用户在同一 Studio 内自由混用 AI、模板和手动参数。
4. 让官方模板包含参数教学，而不只是一个不可解释的效果按钮。
5. 让批量工作流可预览、可解释、可恢复。
6. 为模型、导入、导出和元数据能力提供受控插件扩展面。

### 2.2 非目标

- 不做云端照片托管、账号体系或跨设备同步。
- 不复制或接管用户已有本地照片，除非用户从可移除设备明确执行复制导入。
- 不做评分体系。
- 不在首版做社区模板市场、点赞排行、作者主页或任意第三方代码安装。
- 不允许 AI、模板、技能或插件绕过白盒参数模型直接黑盒改图。
- 不自动删除相机、存储卡或手机中的源文件。
- 本提案不锁定最终字体、色彩、间距和动效；这些属于后续视觉设计收口，但不得改变本文的信息层级与职责边界。

### 2.3 品牌基线

- 对外名称写作 **LookLift**；仓库、包、CLI 与配置键保持 `looklift`，不做破坏性改名。
- 应用图标采用已选定的 A 第一版：几何字母 L 与独立上升调色曲线组合，曲线末端使用直接圆头，
  不增加额外圆点。当前图标与界面图标仍是方向稿，正式发布前统一做矢量、光学和小尺寸适配。

## 3. 设计原则

### 3.1 文件夹是真相

图库镜像用户选择的本地根目录和子目录。照片在所有索引目录之外时，不出现在图库中；LookLift
不制造离线占位，也不要求用户接受专有图库目录结构。

### 3.2 原片不可改写

调色版本只保存白盒参数、预览、AI 摘要和导出记录。原片始终不变，只有导出时才生成新的
JPEG、TIFF、预设或 sidecar 文件。

### 3.3 单一参数真相源

AI 建议、模板套用、自动化步骤和右侧参数控件都生成同一种参数变更，范围与默认值来自核心参数契约。

### 3.4 单一版本历史

AI、模板和手调共用同一套 undo/redo 和版本栈。不存在互不相通的“AI 效果层”和“手动效果层”。

### 3.5 照片优先

进入 Studio 后，全局导航默认折叠为窄图标轨。AI 对话、参数面板和底部照片带也可独立收起，
剩余空间全部交给画布。

### 3.6 本地优先、外部调用明示

索引、缩略图、渲染、版本管理和基础技术分析默认在本地完成。调用外部 AI 前，界面明确显示供应商
以及将发送的照片。

## 4. 全局信息架构

### 4.1 常驻可折叠导航

全局左栏包含：

1. 首页
2. 我的图库
3. 大师模板
4. 自动化技能
5. 插件
6. 设置与帮助

展开时显示图标和文字，折叠后保留约 48–56px 的图标轨。Studio 默认折叠，普通平台页面沿用用户上次状态。

### 4.2 顶部标签

顶部标签保存工作上下文：

- 首页固定存在。
- 打开的图库文件夹可以成为标签。
- 修图会话可以成为标签。
- 标签切换保留照片选择、滚动位置、面板折叠和尚未结束的临时预览。
- `＋` 用于添加文件夹、从设备导入、快速修图或新建修图会话。

### 4.3 顶层页面职责

| 页面 | 职责 |
|---|---|
| 首页 | 三种开始入口、最近图库、继续修图、推荐模板和快捷自动化 |
| 我的图库 | 文件夹树、照片网格、标签搜索、版本与导出记录 |
| 大师模板 | 官方模板和我的模板；预览、套用、参数教学 |
| 自动化技能 | 内置工作流、我的技能、运行配置和执行记录 |
| 插件 | 已安装能力、官方目录、供应商配置和权限 |
| 修图 Studio | AI、画布、白盒参数和本次照片带组成的专业编辑工作台 |

## 5. 首页与导入

### 5.1 首页

首页主视觉使用“今天想修哪组照片？”作为行动入口，提供：

- **添加文件夹**：索引本机现有目录，不复制原文件。
- **从设备导入**：从相机、存储卡或手机预览并选择照片，复制到本地目标目录。
- **快速修图**：直接打开单张或少量照片，不索引所在文件夹，也不进入图库。

首页下方展示最近图库、继续修图、推荐模板和常用自动化。它们是快捷入口，不承担完整管理功能。

### 5.2 添加本机文件夹

用户选择一个或多个根目录。LookLift 增量扫描支持的图片、生成缩略图并建立本地索引，但不复制文件。
用户可在预览网格中选择本次要修的照片并进入 Studio。

### 5.3 从可移除设备复制导入

首次使用时设置长期默认图库根目录，例如“图片/LookLift”。后续导入只需填写批次名，默认生成
“日期 + 批次名”子目录，同时允许临时更换目标。

流程：

1. 检测相机、存储卡或手机并显示来源。
2. 读取缩略图，支持全选、日期筛选、未导入筛选和缩略图大小调整。
3. 用户选择照片；内容指纹标识已导入项，默认跳过重复照片。
4. 文件先复制到目标目录的临时文件。
5. 完成大小或内容校验后原子改名为正式文件。
6. 目标目录已在索引范围内则直接刷新，否则自动添加为索引目录。
7. 导入成功的选中照片组成一个 Studio 会话。

任何情况下都不自动删除设备原片。

RAW 与 JPEG 在导入网格里按真实文件分别展示；即使同名也不强制合并，用户自行选择导入哪一个或两个都导入。
在 v2.3-B 前设置 RAW 可行性门：若完整解码的性能、内存、色彩和高位深链路达标，代理预览与最终导出均从
RAW 解码产生；否则用内嵌 JPEG 做编辑预览，并输出 XMP sidecar 供专业软件完成高画质导出。

## 6. 聊天修图 Studio

### 6.1 四区布局

- **最左全局导航轨**：常驻但默认折叠。
- **左 AI 对话**：本地技术分析、用户消息、AI 参数建议与统一附件入口。
- **中照片画布**：原图/效果分割对比、缩放、适合窗口和临时预览状态。
- **右白盒面板**：直方图、全局强度、基础、HSL、曲线、颜色分级和效果。
- **底部照片带**：只显示本次导入或选择的照片，不显示整个图库。

左 AI、右参数和底部照片带可独立收起。

### 6.2 三种并列修图路径

1. **纯手动**：用户直接调整右侧参数。
2. **AI 协作**：用户描述目标，AI 返回参数 delta 和简短理由。
3. **模板起步**：直接套用模板，或从 AI 输入框 `＋` 将模板作为自适应参考。

三种路径可以在同一版本中连续混用。

### 6.3 AI 分层触发

- 切换照片时自动运行免费的本地技术分析，给出曝光、色彩和可恢复空间等基础判断。
- 完整 AI 修图建议只有在用户点击或发送消息后才调用外部或本地模型。
- 从外部模型发送照片前必须显示供应商和数据范围。

外部模型只接收最长边 2048px、重新编码且无 EXIF 的代理图。默认可附带 ISO、快门、光圈、焦距、曝光补偿、
白平衡和色彩空间等安全白名单文本，用户可以关闭；GPS、时间、序列号、作者、版权、完整路径、文件名、
内部 ID 和 MakerNotes 永不发送。

普通消息只生成一次建议；用户主动点击“AI 精修”时，最多追加两轮。AI 不默认后台自动连跑三轮。

### 6.4 临时预览

AI 建议返回后自动渲染到画布，但先进入临时状态：

- 用户点击“保留此版本”后写入正式版本。
- 用户继续拖动右侧参数，视为接受临时结果并继续手调。
- 用户点击“撤销”则回到上一个正式版本。
- 切换照片或关闭会话时若仍有临时结果，提示保留或放弃。

### 6.5 模板的两种语义

- 在模板页或图库中点击“直接套用”：立即用模板原始参数生成临时预览，可调强度。
- 从 AI 输入框 `＋` 添加模板：模板成为对话附件，AI 根据当前照片自适应后返回新参数。

### 6.6 版本与导出

顶部提供 undo、redo、版本和导出入口。版本可命名、比较、设为当前版本和导出。导出路径记录到当前版本，
但导出文件不会作为新的独立照片卡片再次进入图库。

默认导出到源文件夹下的 `LookLift Exports`，命名为 `<原文件名>-<版本名>.<格式>`；冲突时追加序号，
永不静默覆盖。每个源文件夹记住用户上次选择。RAW sidecar 默认写在原片旁，并在首次操作时说明影响。

## 7. 图库与版本模型

### 7.1 图库布局

- 左侧局部栏：真实索引文件夹树和快捷视图。
- 顶部工具栏：文件名、文件夹、标签、调整状态和格式筛选；不提供评分筛选。
- 主区：照片网格，卡片默认展示当前版本效果。
- 可选右侧详情：原片、版本栈、导出记录、标签和基础元数据。

快捷视图包括最近导入、已调整、已导出和未加标签。

### 7.2 一张照片只出现一次

照片卡片显示：

- 当前版本名称，或“原片”。
- 版本数量。
- 已调整、已导出等状态。
- 用户标签。

原片、其他调色方案与导出记录在详情中的版本栈展开，不平铺成重复照片。

### 7.3 文件移动

- 在任一索引根目录内部移动或改名：文件监听器先用系统文件 ID，再用内容指纹更新路径。
- 移出所有索引根目录：照片立即从图库查询结果消失，不显示离线卡片。
- 以后重新进入任一索引目录：可按内容指纹恢复旧标签与版本。

v2.3 首版不在应用内提供移动或重命名，只提供“在资源管理器中显示”。图库的定位是本地文件夹的增强浏览器，
不是替代文件管理器。

### 7.4 删除语义

- “从图库移除”只取消文件夹索引，不删除原文件。
- “删除原文件”是独立高风险操作，显示完整路径并移入系统回收站。
- 删除调色版本时，确认框列出关联导出文件。
- 仍在登记路径且文件身份匹配的导出文件移入系统回收站，然后删除版本。
- 被用户移动、改名或替换的导出文件只解除关联，不盲目删除。
- 若回收站操作失败，保留版本并列出失败路径，允许重试或只解除关联。

## 8. 模板、自动化与插件

### 8.1 大师模板

首版仅包含：

- LookLift 官方制作的通用风格模板。
- 用户自己收藏、调整或从参考成片逆向得到的模板。

模板卡片包含前后预览、适用场景、白盒参数、强度语义和教学说明。暂不做社区上传、排行和未经授权的
摄影师商业模板；模板采用“柔和胶片、清透人像、青橙旅拍”等通用描述。

### 8.2 自动化技能

自动化是可解释的修图工作流，不是任意脚本执行器。首批能力可包括：

- 批量统一曝光和白平衡。
- 对一组照片自适应套用指定模板。
- 人像选片后统一肤色。
- 按尺寸、格式、水印和命名规则批量导出。
- 将常用步骤保存为用户技能。

每个技能显式声明输入照片、处理步骤、将修改的参数和导出目标。执行前提供抽样预览；失败可按照片重试。

### 8.3 插件

插件只扩展受控能力：

- AI 模型供应商与本地模型。
- 导入来源、导出格式和发布目标。
- 元数据读取和图库连接。
- 白盒分析器、模板包和自动化技能。

首版插件页只做已安装能力和官方插件目录，不开放任意第三方代码安装。安装时显示来源、签名以及文件、网络、
模型权限。插件只能通过受控 API 工作，不能黑盒改写像素。

### 8.4 Studio 统一附件入口

AI 输入框的 `＋` 提供模板、技能和参考图选择器。插件提供底层能力，但普通用户无需理解某项能力来自哪个插件。

## 9. 技术架构与状态所有权

### 9.1 Tauri 桌面外壳

负责窗口、全局标签和导航、系统文件选择器、设备发现、系统回收站、插件权限提示和 Python sidecar 生命周期。

### 9.2 React 应用

负责平台页面、Studio 会话、临时预览、照片选择、面板折叠、进度与错误展示。预览请求必须支持取消，
并用请求序号拒绝过期响应。

### 9.3 Python 本地服务

负责参数契约、唯一渲染引擎、文件夹索引、内容指纹、缩略图、复制导入、版本、导出、本地数据库、AI 供应商、
自动化运行和插件受控 API。

业务逻辑不得在 Tauri 或 React 中重复实现。

## 10. 核心数据模型

| 实体 | 关键数据 |
|---|---|
| `IndexedRoot` | 根目录、显示名、扫描状态、监听配置 |
| `PhotoAsset` | 稳定 ID、当前路径、内容指纹、拍摄元数据、缩略图 |
| `Tag` | 用户标签及照片关联；不含评分 |
| `EditVersion` | analysis、factor、来源、AI 摘要、父版本、当前标记 |
| `ExportRecord` | 版本、输出路径、格式、文件身份、导出时间 |
| `EditSession` | 本次照片集合、当前照片、临时预览、面板状态 |
| `Template` | 白盒参数、强度、适用场景、预览和教学 |
| `Skill` | 显式输入、步骤、参数影响、预览策略和输出 |
| `PluginGrant` | 插件来源、版本、签名和已授予权限 |

标签、版本和导出记录默认存中央本地数据库，避免污染用户照片文件夹。用户可以选择写出 sidecar 进行备份或迁移。

数据库位于应用数据目录，使用事务与 `schema_version`；迁移前轮换保留最近三份备份，并提供手动导出/恢复。
备份不包含原片、导出图和缩略图。数据库损坏时先只读恢复，不直接覆盖原库。

## 11. 关键数据流

### 11.1 文件夹索引

选择根目录 → 增量扫描 → 文件身份与元数据 → 缩略图 → 图库查询。

### 11.2 设备导入

选择照片 → 临时文件复制 → 完整性校验 → 原子落盘 → 增量索引 → Studio 会话。

### 11.3 编辑

AI、模板、技能或手动产生参数 delta → 可取消预览 → 临时状态 → 用户接受 → 版本事务。

### 11.4 导出

读取当前版本 → 临时输出 → 渲染校验 → 原子改名 → 写入 `ExportRecord`。

## 12. 异常处理

| 场景 | 行为 |
|---|---|
| 导入时拔出设备 | 保留已校验文件；未完成项可在重新连接后续传，不生成半文件 |
| 照片在索引目录内部移动 | 按文件 ID 或内容指纹更新路径，版本不丢失 |
| 照片移出索引范围 | 立即从图库消失；后台元数据可供以后恢复 |
| AI 失败或取消 | 正式版本不变；显示重试，手动调参继续可用 |
| 旧预览晚返回 | AbortController 与请求序号共同拒绝，画布不回跳 |
| 导出失败 | 不写 `ExportRecord`；保留临时文件用于诊断或安全清理 |
| 删除导出文件失败 | 版本保留并显示失败路径，允许重试或解除关联 |
| 插件权限不足 | 阻止调用并定位到缺失权限，不自动扩大授权 |

## 13. 响应式桌面布局

LookLift 是桌面应用，不为手机布局妥协专业工作台：

- **≥1440px**：全局轨、AI、画布、参数和照片带可同时显示。
- **1100–1439px**：全局栏默认折叠；聊天和参数面板按用户上次状态显示。
- **900–1099px**：聊天变为抽屉，右参数面板保留紧凑宽度，照片带降低高度。
- **<900px**：提示增大窗口，不继续压缩专业控件。

最低窗口尺寸与各面板最小宽度在对应迭代的视觉/实现 spec 中用真机测量确定，不在本跨迭代提案中预写像素级实现值。

## 14. 测试与验收策略

### 14.1 单元测试

覆盖参数 delta、版本栈、标签、内容指纹匹配、重复检测、导出记录、删除安全校验和插件权限解析。

### 14.2 集成测试

使用临时目录模拟扫描、内部移动、移出、重新进入、设备中断、恢复导入、导出失败和回收站失败。

### 14.3 界面流程测试

使用假 AI 和假文件系统事件覆盖：

首页 → 设备导入 → Studio → AI 临时预览 → 保留 → 手调 → 图库版本 → 导出。

还需覆盖本机文件夹只索引、快速修图不入库、模板直接套用、模板作为 AI 附件和自动化抽样预览。

### 14.4 人工验收

在真实 Windows/Tauri 环境检查：

- 相机或存储卡导入与中断恢复。
- 约 5000 条索引记录下的滚动和标签搜索；首版不为十万级图库设计复杂缓存。
- 40MP 照片内存与预览延迟。
- 多面板折叠与不同窗口宽度。
- AI、模板和手调连续混用的版本语义。
- 模板观感、参数教学和最终图标/字体/颜色体系。

## 15. 已确认的后续拆分

本设计范围过大，不作为一个实施迭代一次完成：

1. **v2.0.0 收口**：v2.0-A/B 完成 M1-M8、统一版本号和发布基线，不新增平台范围。
2. **v2.1 AI Studio**：对话 delta、临时预览、按需精修、主曲线、最小会话/版本持久化与安全代理图。
3. **v2.2 平台外壳**：全局导航、顶部标签、首页三入口、会话恢复和 Studio 迁入。
4. **v2.3-A 本地图库**：Windows/NTFS 索引根、缩略图、标签、当前版本卡片与导出记录。
5. **RAW 可行性门**：在设备导入前决定完整 RAW 解码还是内嵌 JPEG + XMP 路径。
6. **v2.3-B 设备导入**：设备来源、RAW/JPEG 分列选择、默认图库目录、复制校验与重复检测。
7. **v2.4 模板与教学**：官方/用户模板、直接套用、AI 附件与参数讲解。

## 16. 首版规模与平台边界

- 本地图库面向数百至低千张照片，以约 5000 条记录作为性能验证规模，接口保留分页扩展缝隙。
- v2.3 首版 Windows/NTFS 优先；NAS 仅提供手动刷新和尽力而为的监听，macOS/Linux 图库后续实施。
- 当前渲染内部使用 float32，但现有导出最终量化为 8-bit；RAW 门必须验证高位深输入在最终导出前不被提前量化。
- 第一阶段 RAW 画质目标是高质量 8-bit sRGB JPEG；真正 16-bit TIFF 延后，不阻塞设备导入。
