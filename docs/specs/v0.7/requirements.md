## 一句话目标

把 looklift 打包成 Windows 单文件可执行程序,让不装 Python 的摄影爱好者双击即可用,
GUI 首次启动完成配置引导,补齐使用文档。

## 依赖前置版本

| 依赖 | 内容 | 状态 |
|---|---|---|
| v0.4 | pywebview GUI 壳、首次启动配置向导 | 本迭代**只做打包适配**,不改配置向导逻辑本身,假定其已实现 |
| v0.6 | 预置 look 库、`importlib.resources` 资源定位方式 | 打包需要把这批资产一起塞进 exe,复用同一套定位方式(见 [design.md](./design.md) 决策 3) |

## 验收标准

- [ ] Windows 10/11 无 Python 环境的干净机器(或虚拟机)解压/双击后能启动 GUI,跑通 U1/U4/U8 全流程(人工验收)
- [ ] 打包产物内含 GUI 与 CLI 双入口(默认无参数启动 GUI,带 CLI 子命令走命令行)
- [ ] 首次启动引导用户完成 provider 配置(复用 v0.4 配置向导,本迭代只验证打包后能正常跑,不改向导本身)
- [ ] WebView2 缺失时给出中文提示 + 官方下载链接,而不是崩溃或白屏
- [ ] README 新增"下载即用"安装路径,同时保留 `pip install looklift` 路线给技术用户/Mac 用户
- [ ] Windows Defender 对打包产物不误报为病毒,或已有已知误报的应对说明记录在文档中
- [ ] CI 保持双平台(ubuntu/windows)+ macOS 绿,pip 路线不受打包改动影响

## 非目标

- 不做 macOS `.app` 打包(constitution 已定:Mac 走 `pip install looklift`,CI 已覆盖核心兼容性)
- 不做代码签名(成本 $200+/年,与"零运营成本"定位冲突;是否例外由作者决定,列入待决问题)
- 不做自动更新机制,用户重新下载新版本
- 不做安装程序(installer/MSI),只做单文件/单目录的绿色可执行程序
- 不改变 v0.4 配置向导的交互逻辑,只做打包环境下的路径适配
